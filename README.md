# Todo App

シンプルなカンバン形式の Todo アプリです。  
タスクの追加・編集・削除・ステータス変更に加え、ドラッグ＆ドロップによる並び替え・列移動に対応しています。

ポートフォリオ用の作品として、**見た目の統一感** と **保守しやすいコンポーネント設計** を意識して制作しました。

## 概要

このアプリは、日々のタスクを

- Todo
- Doing
- Done

の3つのステータスで管理できる、シンプルなタスク管理アプリです。

単なる CRUD 実装だけでなく、以下のような実用的な操作も行えます。

- タスクの追加
- タスクの編集
- タスクの削除
- ステータス変更
- ドラッグ＆ドロップでの列移動
- ドラッグ＆ドロップでの同一カラム内並び替え
- ローカルストレージへの保存

## 主な機能

### 1. タスクの追加

入力欄から新しいタスクを追加できます。  
追加したタスクは `Todo` カラムに保存されます。

### 2. タスクの編集

既存タスクのタイトルを編集できます。  
編集中は専用UIに切り替わり、保存・キャンセル操作が可能です。

### 3. タスクの削除

不要なタスクを削除できます。

### 4. ステータス変更

セレクトボックスから `Todo / Doing / Done` を切り替えられます。

### 5. ドラッグ＆ドロップ

タスクをドラッグして、

- 別カラムに移動
- 同じカラム内で順序変更

ができます。

### 6. データ保存

タスクは LocalStorage に保存しているため、ページを再読み込みしても内容が保持されます。

## 工夫した点

### 1. 保守しやすい分割

当初は1ファイルに処理をまとめていましたが、責務が大きくなってきたため、以下の観点で分割しました。

- フォーム
- カラム
- タスクカード
- 画面全体ロジック

これにより、見通しがよくなり、今後の機能追加もしやすい構造になっています。

### 2. 型安全を意識した実装

TypeScript を使い、`status` などの値を明確に型定義しました。  
イベント引数や状態更新時も、できるだけ unsafe な書き方を避けています。

### 3. 並び順の保持

ドラッグ＆ドロップによる順序変更を実現するため、各タスクに `position` を持たせています。  
表示順だけでなく保存順も意識して実装しました。

### 4. UI崩れへの対応

入力欄とボタンの干渉、編集中のレイアウト崩れなど、実装途中で発生した問題に対して、`grid` や `min-width: 0` を用いて安定したレイアウトに修正しました。

## 実装上のポイント

### 状態管理

Pinia を用いて、Todo 配列や更新状態を管理しています。

### 永続化

バックエンドは使わず、LocalStorage に保存しています。  
そのため、軽量に動作確認できる構成になっています。

### ドラッグ＆ドロップ

HTML5 の Drag and Drop API を利用して実装しました。  
ライブラリに頼らず、基本的なイベント処理を自前で組んでいます。
